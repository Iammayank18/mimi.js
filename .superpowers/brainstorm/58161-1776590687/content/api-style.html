How should route handlers look in mimijs?

This is the most visible API decision — it affects every developer who uses the framework.

A

Keep req / res / next (Express-compatible)

Identical surface to Express. Every Express middleware, tutorial, and Stack Overflow answer works unchanged. TypeScript types are enriched (Zod-inferred body, typed params) but the calling convention is the same.

mimijs handler — Option A
app.post('/users', validate({ body: z.object({ name: z.string() }) }), async (req, res) => { // req.body is typed: { name: string } const user = await db.create(req.body); res.status(201).json(user); } );

Wins

  • Zero migration cost from Express
  • All existing middleware works
  • No new concepts

Costs

  • req/res are prototype-patched globals — hard to tree-shake
  • TypeScript inference is harder to wire
B

Context object ctx (modern, Hono-style)

A single ctx object per request carries everything: ctx.req, ctx.res, helper methods, and Zod-validated typed data. Cleaner TypeScript generics, easier testing.

mimijs handler — Option B
app.post('/users', validate({ body: z.object({ name: z.string() }) }), async (ctx) => { // ctx.body is typed: { name: string } const user = await db.create(ctx.body); return ctx.json(user, 201); } );

Wins

  • Cleaner TS generics — types flow naturally
  • Easier to mock in tests (one object)
  • Return-based responses (no forgetting res.send)

Costs

  • Express middleware needs an adapter
  • New concept for Express developers
C

Dual mode — both, with compatibility bridge

The core runs on ctx internally. Route handlers can use either style — mimijs detects the arity and adapts. Express middleware mounts via app.use(expressMiddleware()) and works unchanged.

mimijs handler — Option C
// New style (preferred) app.post('/users', async (ctx) => { const user = await db.create(ctx.body); return ctx.json(user, 201); }); // Express style still works — zero migration friction app.post('/users', async (req, res) => { const user = await db.create(req.body); res.status(201).json(user); });

Wins

  • Best of both: migrate gradually, write new code cleanly
  • Strongest story for Express migrants
  • TypeScript can infer which style from function signature

Costs

  • Two mental models in one codebase
  • More implementation complexity internally

Pick one — this defines how every mimijs tutorial and example will look.
